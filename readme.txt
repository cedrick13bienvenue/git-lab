Git Lab - Practice Repository

This repository is used to practice core Git workflows, including:
- Branching and merging
- Staging and committing changes
- Pushing to remote repositories
- Opening pull requests for code review
- Resolving merge conflicts

Getting Started
---------------
1. Clone the repository
2. Create a new branch for your feature or fix
3. Make your changes and commit them with clear messages
4. Push your branch and open a pull request
5. Request a review and address feedback

Commit Message Guidelines
--------------------------
- Use the imperative mood: "Add feature" not "Added feature"
- Keep the subject line under 50 characters
- Explain *why* in the body if the change is non-obvious
- Reference related issues when applicable (e.g. "Closes #12")

Branch Naming Conventions
--------------------------
- feature/<short-description>
- fix/<short-description>
- chore/<short-description>
- docs/<short-description>

Pull Request Guidelines
------------------------
- Keep PRs small and focused on a single change
- Write a clear description of what and why
- Link to any relevant issues or tickets
- Ensure all checklist items are completed before requesting review

Notes
------
- Always pull the latest changes from main before creating a new branch
- Delete branches after they are merged to keep the repo clean
